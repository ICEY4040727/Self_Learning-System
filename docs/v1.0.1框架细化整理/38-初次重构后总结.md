# v1.0.1 重构后架构总结

## 一、系统定位

**苏格拉底学习系统 (Socratic Learning System)**

核心目标：通过苏格拉底式对话（启发式提问）引导用户自主思考，而非直接灌输答案。

## 三、核心数据模型

### 3.1 用户与身份

| 模型 | 说明 |
|------|------|
| `User` | 系统用户，含 API Key（用于调用 LLM） |
| `UserProfile` | 跨世界全局画像（展示用） |

### 3.2 世界与课程

| 模型 | 说明 |
|------|------|
| `World` | 用户创建的学习世界（主题） |
| `Course` | 世界内的课程/科目 |

### 3.3 角色系统 (Character)

**Phase 1.5 DD1: TeacherPersona 已合并到 Character**

| 角色类型 | 说明 | 在 Session 中的关联 |
|----------|------|-------------------|
| `sage` | AI 导师/老师 | `Session.sage_character_id` |
| `traveler` | 旅人角色（用户化身） | `Session.traveler_character_id` |

**Character 表字段:**
- `name`: 角色名
- `type`: "sage" 或 "traveler"
- `system_prompt_template`: 自定义 prompt 模板
- `template_name`: 模板标识 (socrates/einstein...)
- `traits`: 性格参数 {strictness, pace, questioning, warmth, humor}
- `is_active`: 是否可用于教学

### 3.4 学习会话 (Session)

| 字段 | 说明 |
|------|------|
| `course_id` | 关联课程 |
| `world_id` | 关联世界 |
| `sage_character_id` | AI 导师 |
| `traveler_character_id` | 用户化身 |
| `learner_profile_id` | 学习档案 |
| `relationship` | JSON: {stage, dimensions: {trust, familiarity...}} |
| `started_at` / `ended_at` | 会话时间 |
| `parent_checkpoint_id` | 从存档恢复时指向原存档 |
| `branch_name` | 分支名 |

### 3.5 聊天记录 (ChatMessage)

| 字段 | 说明 |
|------|------|
| `session_id` | 所属会话 |
| `sender_type` | "user" 或 "teacher" |
| `content` | 消息内容 |
| `emotion_analysis` | 情感分析结果 JSON |
| `used_memory_ids` | 本次回复使用了哪些记忆 |

### 3.6 记忆事实 (MemoryFact)

**核心创新：让 AI 记住用户的学习状态**

| 字段 | 说明 |
|------|------|
| `character_id` | 指向 sage character（AI 老师） |
| `world_id` | 可为 NULL（跨世界事实） |
| `fact_type` | 类型: student_state/concept_struggle/concept_mastered/preference/event/commitment |
| `content` | 记忆内容（≤500字） |
| `concept_tags` | 关联概念标签 |
| `salience` | 重要度 0.1-1.0 |
| `recall_count` | 被检索次数 |
| `expires_at` | 过期时间 |

### 3.7 学习档案 (LearnerProfile)

| 字段 | 说明 |
|------|------|
| `user_id` + `world_id` | 组合唯一 |
| `profile` | JSON: 学习偏好、元认知数据 |
| `session_count` | 会话计数 |

### 3.8 存档 (Checkpoint)

| 字段 | 说明 |
|------|------|
| `world_id` | 关联世界 |
| `session_id` | 关联会话（可选） |
| `save_name` | 存档名 |
| `message_index` | 存档点的消息索引 |
| `state` | JSON: 关系状态、课程ID等 |
| `file_path` | JSON 文件路径（Issue #207） |
| `file_size_bytes` | 文件大小 |

## 五、核心服务层

### 5.1 LearningEngine (`learning_engine.py`)

**消息处理流水线:**

```
process_message(user_message)
    │
    ├─1. 获取会话上下文 (Session)
    │       - sage_character (AI 导师)
    │       - traveler_character (用户化身)
    │       - learner_profile (学习档案)
    │
    ├─2. 情感分析 (DynamicAnalyzer.analyze_emotion)
    │       - 分析用户消息的情感
    │       - 返回 emotion_type, valence, arousal
    │
    ├─3. 构建提示词 (PromptBuilder.build)
    │       ├─ RelationshipModule: 注入关系阶段指令
    │       ├─ MemoryFactsModule: 检索相关记忆
    │       └─ 注入 system_prompt_template
    │
    ├─4. 调用 LLM (get_llm_adapter.chat)
    │       - 支持 Claude/Gemini 等
    │
    ├─5. 解析回复 (parse_tool_request)
    │       - 解析 <tool> 标签（预留）
    │
    ├─6. 提取记忆 (memory_extractor.extract)
    │       - 从 LLM 回复中提取 <memory> 标签
    │       - 写入 MemoryFact 表
    │
    ├─7. 更新关系 (relationship_service.update)
    │       - 根据情感更新关系维度
    │       - 触发关系阶段升级
    │
    └─8. 返回 ChatResponse
            - type: text/choice
            - reply: AI 回复
            - emotion: 情感分析
            - relationship_events: 关系变化事件
            - memory_extracted_count: 本次提取的记忆数
```

### 5.2 PromptBuilder (`prompt_builder/`)

**模块化提示词构建器:**

| 模块 | 功能 |
|------|------|
| `RelationshipModule` | 根据关系阶段注入不同教学风格指令 |
| `MemoryFactsModule` | 检索与当前话题相关的记忆，注入上下文 |
| `SceneConfig` | 不同场景 (LEARNING/EXPLORATION) 的模板配置 |

### 5.3 MemoryFactsService (`memory_facts.py`)

**记忆生命周期:**

```
┌─────────────┐
│ Seed Memory │ 首次会话时，从 traveler_character + learner_profile 生成
└──────┬──────┘
       ▼
┌─────────────────┐
│ 每轮对话后提取   │ 从 LLM 回复解析 <memory> 标签
└──────┬──────────┘
       ▼
┌─────────────────┐
│ 记忆写入        │ 存储: content, fact_type, salience...
└──────┬──────────┘
       ▼
┌─────────────────┐
│ 记忆检索        │ 根据当前话题检索相关记忆
└──────┬──────────┘
       ▼
┌─────────────────┐
│ 注入下一轮对话  │ 作为上下文传给 LLM
└─────────────────┘
```

### 5.4 RelationshipService (`relationship.py`)

**关系系统:**

| 关系阶段 | 标签 | 说明 |
|----------|------|------|
| stranger | 初识 | 默认状态 |
| acquaintance | 相识 | 建立基础信任 |
| friend | 朋友 | 开始深入交流 |
| mentor | 导师 | 高阶启发式提问 |
| partner | 伙伴 | 协作探索 |

**关系维度:** trust, familiarity, respect, comfort

### 5.5 DynamicAnalyzer (`dynamic_analyzer.py`)

**情感分析:** 分析用户消息的情感类型、效价、唤醒度

**学习分析:** 评估理解度、进度

## 六、学习促进机制

### 6.1 苏格拉底式提问

AI 导师根据关系阶段调整提问策略：

| 阶段 | 策略 |
|------|------|
| 初识 | 直接讲解为主 |
| 相识 | 简单引导 |
| 朋友 | 启发式提问增多 |
| 导师 | 深度追问，挑战假设 |
| 伙伴 | 协作式讨论 |

### 6.2 记忆驱动个性化

```
用户: "我不太理解递归"
         │
         ▼
MemoryFactsService 检索:
- concept_struggle: "递归" (salience=0.9)
- student_state: "偏好通过代码示例理解"
         │
         ▼
PromptBuilder 注入:
- 记忆: 对方在递归上有困难
- 偏好: 需要代码示例
         │
         ▼
AI 回复: "递归确实是个难点。让我用你熟悉的求和来演示..."
```

### 6.3 情感感知

```
用户消息 → DynamicAnalyzer.analyze_emotion
         │
         ├─ emotion_type: confusion/frustration/satisfaction...
         ├─ valence: 0.0-1.0 (正负)
         ├─ arousal: 0.0-1.0 (激活度)
         │
         ▼
影响:
- 关系维度更新 (困惑→降低 comfort, 满足→提高 trust)
- AI 表情提示 (expression_hint)
- 学习节奏调整
```

## 七、存档与分支机制

```
Session A ──┬── Checkpoint 1

```

## 九、总结

### 系统优势

1. **记忆系统**: 通过 MemoryFact 让 AI 真正"认识"用户
2. **关系系统**: 渐进式师生关系，影响教学风格
3. **情感感知**: 实时感知用户情绪，调整策略
4. **存档分支**: 支持学习路径探索
5. **模块化提示词**: 清晰的关注点分离

### 需完善

1. FSRS 间隔重复算法集成
2. 工具调用机制设计（是否需要）
3. UserProfile 与 LearnerProfile 的边界
4. SessionLocal 自管连接清理

# 问题 22: 前端页面背景图片硬编码导致视觉不统一

## ⚠️ 未解决

**说明**: 各页面仍使用不同的背景图片，未统一

## 问题类型
前端硬编码 / 视觉不一致 / UX 问题

## 涉及文件
- `frontend/src/views/Login.vue`（`login-bg.jpg`，absolute）
- `frontend/src/views/Home.vue`（`home-bg.png`，absolute）
- `frontend/src/views/Worlds.vue`（`home-bg.png`，fixed）
- `frontend/src/views/WorldDetail.vue`（`char-bg.jpg`，fixed）
- `frontend/src/views/Character.vue`（`char-bg.jpg`，fixed）
- `frontend/src/views/CoursePage.vue`（`char-bg.jpg`，fixed）
- `frontend/src/views/Learning.vue`（动态 `store.sceneBackground`，absolute）
- `frontend/src/views/Archive.vue`（Unsplash 外链，absolute）
- `frontend/src/views/Settings.vue`（Unsplash 外链，fixed）
- `frontend/src/App.vue`（`page-fade` Transition 缺少 `mode`）

## 子问题 A：背景图片来源不一致

| 页面 | 图片来源 | 加载方式 |
|------|---------|---------|
| Login | `@/assets/login-bg.jpg` | Vite import（本地打包） |
| Home | `@/assets/home-bg.png` | Vite import（本地打包） |
| Worlds | `@/assets/home-bg.png` | Vite import（本地打包） |
| WorldDetail | `@/assets/char-bg.jpg` | Vite import（本地打包） |
| Character | `@/assets/char-bg.jpg` | Vite import（本地打包） |
| CoursePage | `@/assets/char-bg.jpg` | Vite import（本地打包） |
| Learning | 动态 API 返回 | 运行时加载 |
| Archive | `https://images.unsplash.com/photo-167537...?w=1920&q=80` | 外部 URL |
| Settings | `https://images.unsplash.com/photo-150690...?w=1920&q=80` | 外部 URL |

**风险**：
1. Archive 和 Settings 依赖 Unsplash 外链，离线/网络受限时无背景图
2. 外链图片加载慢于本地打包图片，页面打开时会有白底闪烁
3. 没有统一的背景管理策略，新增页面时开发者不知道该用哪种方式

## 子问题 B：`.scene-bg` 定位方式不一致

```css
/* Login, Home, Learning, Archive 使用 absolute */
.scene-bg { position: absolute; inset: 0; }

/* Worlds, WorldDetail, Character, CoursePage, Settings 使用 fixed */
.scene-bg { position: fixed; inset: 0; }
```

**差异**：`absolute` 随页面滚动，`fixed` 固定在视口。两种方式在短页面效果一致，但在长页面（如 WorldDetail 内容很多时）表现不同。

## 子问题 C：页面切换时背景跳变（用户实际看到的问题）

`App.vue` 使用全局 `page-fade` Transition：
```vue
<Transition name="page-fade">
  <component :is="Component" />
</Transition>
```

没有设置 `mode="out-in"`，意味着**新旧页面同时渲染**（cross-fade）。

当从 Worlds（`home-bg.png`）导航到 WorldDetail（`char-bg.jpg`）时：
1. 两个 `.scene-bg` 都是 `position: fixed`，叠加在同一位置
2. 旧页面 opacity 从 1→0，新页面 opacity 从 0→1
3. **用户看到背景从一张图"跳变"到另一张图**
4. 加上 `.scene-bg` 的 `opacity: 0.5`，底层 `--bg-primary` 也会透出，产生三层叠加

## 子问题 D：`scene-overlay` 渐变未统一

每个页面各自定义不同的渐变参数：

```css
/* Home.vue — 紫金色调 */
background: linear-gradient(to bottom, rgba(10,10,30,0.2) 0%, rgba(0,0,0,0.4) 100%)

/* WorldDetail.vue — 金色高光 */
background: radial-gradient(ellipse at 50% 0%, rgba(10,10,30,0.15)...),
            radial-gradient(ellipse at 30% 55%, rgba(255,215,0,0.05)...),
            linear-gradient(to bottom, rgba(10,10,30,0.25)..., rgba(0,0,0,0.45)...)

/* Login.vue — 暗蓝色调 */
background: linear-gradient(135deg, rgba(10,10,30,0.3), rgba(26,26,64,0.4))
```

没有使用 CSS 变量或 mixin，新增页面时需要手动复制和微调。

## 影响分析

1. **用户体验差**：页面切换时背景闪烁/跳变，不连贯
2. **维护困难**：背景逻辑分散在 9 个文件中，无统一管理
3. **视觉不统一**：不同页面使用不同图片、不同定位、不同渐变，缺乏一致性
4. **离线不可靠**：Archive/Settings 依赖外链图片

## 建议修复方向

### 短期修复（立即可做）
1. `App.vue` 的 `<Transition>` 添加 `mode="out-in"`，消除 cross-fade 背景叠加
2. Archive/Settings 的外链图片下载到 `@/assets/` 并改为本地 import

### 中期优化
1. 提取背景配置为统一的 `composables/usePageBackground.ts`
2. 将 `scene-bg` + `scene-overlay` 提升到全局 layout，各页面只控制前景内容
3. 统一所有 `.scene-bg` 使用 `position: fixed`

### 长期架构

背景图由后端世界配置驱动，前端根据路由动态切换。

#### 1. 后端模型扩展

**当前**：`World.background` 字段已存在但未被利用

```python
# backend/models/models.py
class World(Base):
    # ... 现有字段
    background = Column(Text, nullable=True)  # ✅ 已存在，可直接使用
```

**扩展方案**：在 `World` 模型中添加更多背景配置字段：

```python
class World(Base):
    # 基础背景
    background = Column(Text, nullable=True)  # 主背景图 URL
    background_type = Column(String(20), default="image")  # image | gradient | solid
    background_blur = Column(Integer, default=0)  # 模糊度 0-20
    background_overlay = Column(Text, nullable=True)  # 渐变叠加配置 JSON
    
    # 动态背景（可选）
    background_learning = Column(Text, nullable=True)  # 学习场景背景
    background_archive = Column(Text, nullable=True)  # 归档场景背景
```

#### 2. API 扩展

在获取 World/WorldDetail 时返回背景配置：

```python
# backend/api/routes/archive.py 或新建 background.py
@router.get("/worlds/{world_id}/background")
async def get_world_background(
    world_id: int,
    scene: str = "default",  # default | learning | archive
    db: Session = Depends(get_db)
):
    """获取世界背景配置"""
    world = db.query(World).filter(World.id == world_id).first()
    if not world:
        raise HTTPException(404, "World not found")
    
    return {
        "url": getattr(world, f"background_{scene}", None) or world.background,
        "type": world.background_type,
        "blur": world.background_blur,
        "overlay": json.loads(world.background_overlay) if world.background_overlay else None
    }
```

#### 3. 前端统一背景层

创建 `AppBackground.vue` 全局组件：

```vue
<!-- frontend/src/components/AppBackground.vue -->
<template>
  <Teleport to="body">
    <Transition name="bg-fade" mode="out-in">
      <div 
        :key="backgroundKey"
        class="app-background"
        :style="backgroundStyle"
      />
    </Transition>
  </Teleport>
</template>

<script setup lang="ts">
import { computed, watch } from 'vue'
import { useRoute } from 'vue-router'
import { useWorldStore } from '@/stores/worldStore'

const route = useRoute()
const worldStore = useWorldStore()

const backgroundKey = computed(() => 
  `${route.name}-${worldStore.currentWorldId}`
)

const backgroundStyle = computed(() => {
  const bg = worldStore.currentBackground
  if (!bg) return { background: 'var(--bg-primary)' }
  
  return {
    backgroundImage: `url(${bg.url})`,
    backgroundSize: 'cover',
    backgroundPosition: 'center',
    filter: bg.blur ? `blur(${bg.blur}px)` : 'none',
    // 渐变叠加...
  }
})
</script>
```

#### 4. 路由与 Store 集成

```typescript
// frontend/src/router/index.ts
const routes = [
  {
    path: '/world/:id',
    component: WorldDetail,
    meta: { requiresBackground: true, scene: 'default' }
  },
  {
    path: '/world/:id/learn',
    component: Learning,
    meta: { requiresBackground: true, scene: 'learning' }
  },
]

// frontend/src/stores/worldStore.ts
export const useWorldStore = defineStore('world', {
  state: () => ({
    currentWorld: null as World | null,
    currentBackground: null as BackgroundConfig | null,
  }),
  
  actions: {
    async fetchBackground(worldId: number, scene: string = 'default') {
      const res = await api.get(`/worlds/${worldId}/background?scene=${scene}`)
      this.currentBackground = res.data
    },
    
    async setWorld(worldId: number) {
      // 加载世界数据时同时加载背景
      const world = await this.fetchWorld(worldId)
      await this.fetchBackground(worldId, 'default')
    }
  }
})
```

#### 5. App.vue 简化

移除各页面重复的背景代码，只保留内容区域：

```vue
<!-- App.vue -->
<template>
  <AppBackground />  <!-- 全局背景层 -->
  
  <div class="app-content">
    <router-view v-slot="{ Component }">
      <Transition name="page-fade" mode="out-in">
        <component :is="Component" />
      </Transition>
    </router-view>
  </div>
</template>

<style>
.app-content {
  position: relative;  /* 覆盖在背景层上 */
  z-index: 1;
}
</style>
```

#### 6. 页面组件简化

各页面移除背景相关代码，只需处理业务逻辑：

```vue

<!-- 简化后的 WorldDetail.vue -->
```

#### 7. 迁移步骤

| 阶段 | 内容 | 风险 |
|------|------|------|
| Phase 1 | 创建 `AppBackground.vue` + `useBackground` composable | 低 |
| Phase 2 | World API 添加背景端点 | 低 |
| Phase 3 | App.vue 集成 `AppBackground` | 中（影响全局） |
| Phase 4 | 逐页面移除本地背景代码 | 低 |
| Phase 5 | 添加 World 后台编辑背景功能 | 中 |

#### 8. 收益

- **统一管理**：背景逻辑集中在一处
- **平滑过渡**：全局 Transition 实现页面切换动画
- **可配置**：后台可动态调整世界背景
- **离线可靠**：背景 URL 可本地化
- **开发体验**：新增页面无需复制背景代码

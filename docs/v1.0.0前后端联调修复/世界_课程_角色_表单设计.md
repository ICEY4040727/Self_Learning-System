# 世界 / 课程 / 角色 表单设计

## Context

用户在构建 Galgame 风格的学习系统，三层结构 **世界 (World) → 课程 (Course) → 角色 (Character: 旅者 traveler / 知者 sage)**。后端 SQLAlchemy 模型与 API 已基本就绪（见 [models.py](backend/models/models.py)、[archive.py](backend/api/routes/archive.py)），现有前端表单零散且偏"开发者填字段"，缺少**心理学动机设计**与**Galgame 沉浸感**。本次目标是从交互/心理/游戏化角度，设计三张表单的字段、分步流程与文案，作为后续前端实现的依据，

## 设计原则（心理学 + 游戏化）

1. **自我决定理论 (SDT)** — 三大内在动机：自主感 (Autonomy)、胜任感 (Competence)、归属感 (Relatedness)。每张表单都应让用户感到"这是我的世界 / 我在变强 / 我和角色有连接"。
2. **渐进式披露 (Progressive Disclosure)** — 不要一次抛 20 个字段。分 2–3 步，每步 ≤5 个必填，其余折叠为"高级选项"。
3. **叙事化文案** — 字段标题用故事性语言，而非数据库字段名。例："为这个世界命名" 而非 "name"。
4. **具象化选择 > 自由文本** — 用预设卡片 / 标签 / 滑块降低冷启动焦虑（空白文本框是最大杀手），但保留"自定义"入口。
5. **即时反馈** — 每一步实时预览（世界封面卡 / 角色立绘 / 课程胶囊），让用户看见"我创造的东西"。
6. **承诺升级 (Commitment Escalation)** — 第一步只问最轻的问题（名字 + 一张图），让用户先"投入"，再逐步要求更多细节。

## 一、世界 World 表单

**心理目的**：建立**自主感 + 沉浸氛围**。世界是"容器"，应像选一本要写的书的封面。
**复用模型字段**：`name, description, scenes (JSON)`（见 [models.py](backend/models/models.py)）

### Step 1 · 命名你的世界（必填）
| 字段 | 控件 | 文案 / 备注 |
|---|---|---|
| `name` | 单行输入 | "这个世界叫什么？" placeholder："雾港学院 / 星轨图书馆 / 我的代码修炼场" |
| `theme_preset` | 6 张封面卡选择 | 学院 / 图书馆 / 实验室 / 山林书院 / 赛博空间 / 空白画布。每张图带主色调 → 自动写入 `scenes.background` 与主题色 |

> 心理学：第一步只 2 个字段 → 极低门槛 → 触发承诺升级。封面卡比"上传图片"友好 10 倍。

### Step 2 · 这个世界的氛围（可跳过）
| 字段 | 控件 | 备注 |
|---|---|---|
| `description` | 多行文本 (≤140 字) | "用一句话描述这里发生的故事" placeholder："这里是一所漂浮在云端的学院……" |
| `mood_tags` | 多选标签芯片 | 温暖 / 冷峻 / 神秘 / 治愈 / 严肃 / 幽默 / 童话 / 赛博。**写入 `scenes.mood`**，后续注入 system prompt 影响知者语气 |
| `bgm_preset` | 4 个音乐胶囊 (可试听) | 自习室白噪 / 雨夜钢琴 / 晨光木吉他 / 静音。写入 `scenes.bgm` |

### Step 3 · 预览与创建
- 实时合成"世界卡片"（封面 + 名字 + 标签）
- CTA 按钮文案："**进入这个世界**"（不是"创建"——动词换情境化语言）

## 二、课程 Course 表单

**心理目的**：建立**胜任感**。课程是"目标合约"，须把模糊愿望（"我想学 Python"）翻译成可衡量的进度感。
**复用模型字段**：`name, description, target_level`（见 [models.py](backend/models/models.py)）

### Step 1 · 你想掌握什么？（必填）
| 字段 | 控件 | 备注 |
|---|---|---|
| `name` | 单行输入 + 智能建议 | "课程名" placeholder："微积分入门 / 唐诗鉴赏 / Rust 异步编程" |
| `domain_preset` | 8 张图标卡 | 数学 / 语言 / 编程 / 艺术 / 历史 / 科学 / 哲学 / 自定义。决定默认图标 + 配色 |

### Step 2 · 你的起点和目标（核心 — 心理契约）
| 字段 | 控件 | 心理依据 |
|---|---|---|
| `current_level` | 4 段 Likert：完全陌生 / 听说过 / 用过一点 / 已有基础 | 锚定起点 → 后续进度才有"对比感" |
| `target_level` | 4 段："看懂入门 / 能独立应用 / 能教别人 / 达到专业水准" | 复用 `target_level` 字段；明确终点 → 胜任感 |
| `motivation` | 单选卡 + 自定义 | 兴趣 / 考试 / 工作 / 解决具体问题 / 陪孩子学 → 写入 `description` 或扩展字段；**注入 system prompt 让知者懂"为什么"** |

> 游戏化：起点+终点 = 进度条两端。后续学习页可显示 `current → target` 的旅程地图。

### Step 3 · 学习节奏（可跳过 / 折叠"高级"）
| 字段 | 控件 |
|---|---|
| `pace` | 滑块：佛系 / 正常 / 冲刺 |
| `weekly_minutes` | 数字 / 预设按钮：30 / 90 / 180 分钟/周 |
| `description` | 多行文本（自动用前面选项预填一段话，可改） |

> 自动预填降低空白焦虑，同时保留自主感。

### Step 4 · 选择知者（关联角色）

- 从该世界已有的 `sage` 角色中选 1–N 位（多选卡片，显示头像 + 一句话简介）
- 若没有，引导："**先创建一位老师 →**" 跳到角色表单

## 三、角色 Character 表单（旅者 / 知者）

**心理目的**：建立**归属感**。这是 Galgame 的灵魂——角色不是 NPC，是"会和我一起成长的人"。
**复用模型**：`Character (name, type, avatar, personality, background, speech_style, sprites)` + `RolePersonality (traits, system_prompt_template)`，已有 [StepCreateModal.vue](frontend/src/components/character/StepCreateModal.vue)。本设计是对其的**字段层建议**。

> **命名说明**：`TeacherPersona` 已重命名为 `RolePersonality`（角色人格），以更直观地表达其职责——定义角色如何被 AI 扮演。

### 通用 Step 0 · 选择身份
> 大卡片二选一：
> - 🧭 **旅者 (Traveler)** — "这是我自己。代表我在世界里探索的化身。"
> - 📖 **知者 (Sage)** — "一位老师 / 同伴 / 灵感来源，会陪我学习。"

### A. 旅者 Traveler（更轻量，3 字段足矣）
**心理依据**：旅者是用户自我投射，过度配置反而破坏代入。给一个"轻松创建一个我"的体验。

| Step | 字段 | 控件 | 备注 |
|---|---|---|---|
| 1 | `name` | 单行 | "你想叫自己什么？"（可与登录名不同 → 自主感） |
| 1 | `avatar` | 上传 / 9 张预设头像 | 预设比上传重要 10 倍 |
| 2 | `tags` | 多选标签（≤4） | 好奇 / 谨慎 / 急性子 / 慢热 / 视觉型 / 听觉型 / 完美主义 / 半途而废星人。**注入 prompt 让知者知道怎么对你** |
| 2 (可选) | `background` | 多行 (≤80 字) | "想让老师知道你的什么？" placeholder："是一名设计师，三十岁开始学编程……" |

> 不需要 `personality / speech_style` —— 旅者不需要"演"。

### B. 知者 Sage（核心，分 4 步 — 这是"造一个人"）

#### Step 1 · 灵感来源（降低冷启动）
**三个入口卡**：
1. 🎲 **从模板开始** — 严师 / 益友 / 学长 / 神秘导师 / 自由奔放派 / 沉默观察者（每张卡带头像 + 一句台词预览）
2. 🎬 **从角色启发** — 输入"像哈利波特里的卢平教授"→ 调用 `POST /persona/generate`（已存在但未接前端）
3. ✍️ **完全自定义**

> 心理学：选项过多会瘫痪 (Paradox of Choice)。用模板降低门槛，用 AI 生成给"想要独特"的用户出口。

#### Step 2 · 外观与名片
| 字段 | 控件 |
|---|---|
| `name` | 单行 |
| `title` | 单行 placeholder："雾港学院首席研究员" |
| `avatar` + `sprites` | 上传 / 默认立绘库（多表情：默认 / 微笑 / 思考 / 严肃 / 鼓励） |
| `theme_color` | 6 色调色板 → 用于对话框边框、名字色（Galgame 经典做法） |

#### Step 3 · 性格与教学风格（核心 — 决定 AI 行为）
> **关键**：每个滑块的两端都要给**具体行为示例**而非抽象词。

| 字段（→ RolePersonality.traits） | 控件 | 两端示例 |
|---|---|---|
| 严厉度 | 滑块 0–10 | 0 "答错也鼓励你慢慢来" ↔ 10 "答错会直接指出并要求重做" |
| 节奏 | 滑块 | "慢工细活、反复确认" ↔ "快速推进、信息密集" |
| 提问 vs 讲授 | 滑块 | "苏格拉底式提问引导" ↔ "直接讲解+举例" *(默认偏左以契合项目核心)* |
| 情感温度 | 滑块 | "克制冷静" ↔ "热情外放" |
| 幽默感 | 滑块 | "一本正经" ↔ "段子手" |
| `speech_style` | 多选标签 | 文白夹杂 / 口语化 / 学术腔 / 用比喻 / 爱讲故事 / 偶尔吐槽 |

> 滑块写入 `traits` JSON，`speech_style` 直接写字段。**实时预览**："以这种性格，TA 在你答错时可能会说……"（用一段示例对话渲染）—— 这是即时反馈的核心。

#### Step 4 · 背景故事（可跳过 / 折叠）
| 字段 | 控件 |
|---|---|
| `background` | 多行文本 (≤200 字) | placeholder："曾在……研究……，因为……来到这个世界。" |
| `personality` | 多行 (≤120 字，可由前面滑块自动生成草稿) |

> 自动生成草稿 = 降低空白焦虑 + 用户可改 = 保留自主感。

#### Step 5 · 预览与首次问候

- 渲染完整的 Galgame 对话框：立绘 + 名字 + 主题色 + 一句 AI 生成的"初次见面"台词
- CTA："**让 TA 进入你的世界 →**"

   规则：
   - 若 inspiration_type=character 且涉及版权角色（如卢平教授），不要直接复述原作设定，要做"风格借鉴"

4. **路由实现调整** — [archive.py:1407-1456](backend/api/routes/archive.py#L1407-L1456)

## 附录 A · Course.meta JSON 结构与枚举值

前端表单写入、后端 `prompt_builder/modules/course_intent.py` 读取均以此结构为准。除 `current_level / target_level` 外的字段均允许 `null`。

```json
{
  "domain": "programming",
  "current_level": "novice",
  "target_level": "applier",
  "motivation": "work",
  "motivation_detail": "下个月接手团队的 Rust 项目",
  "pace": "normal",
  "weekly_minutes": 90,
  "sage_ids": [12, 17]
}
```

**字段枚举**

| 字段 | 类型 | 枚举值 | 中文标签 | 用途 |
|---|---|---|---|---|
| `domain` | string | `math` / `language` / `programming` / `art` / `history` / `science` / `philosophy` / `custom` | 数学/语言/编程/艺术/历史/科学/哲学/自定义 | 前端图标&配色；prompt 不直接用 |
| `current_level` | string | `none` / `heard` / `tried` / `basic` | 完全陌生/听说过/用过一点/已有基础 | prompt 锚定起点 |
| `target_level` | string | `understand` / `applier` / `teacher` / `expert` | 看懂入门/能独立应用/能教别人/达到专业水准 | prompt 锚定终点 |
| `motivation` | string | `interest` / `exam` / `work` / `problem` / `companion` | 兴趣/考试/工作/解决具体问题/陪孩子学 | prompt 注入"为何而学" |
| `motivation_detail` | string \| null | 自由文本 ≤120 字 | — | 给 LLM 更具体语境 |
| `pace` | string | `chill` / `normal` / `sprint` | 佛系/正常/冲刺 | 影响知者推进速度 |
| `weekly_minutes` | int \| null | 30 / 90 / 180 或自定义 | — | 仅展示，第一版不参与 prompt |
| `sage_ids` | int[] | 关联 `Character.id` | — | 第一版多对多简化方案，开始 session 时让用户选一个 |

**后端建议**

- 在 `backend/services/prompt_builder/modules/course_intent.py` 顶部定义标签映射：
  ```python
  CURRENT_LEVEL_LABELS = {"none": "完全陌生", "heard": "听说过", "tried": "用过一点", "basic": "已有基础"}
  TARGET_LEVEL_LABELS = {"understand": "看懂入门", "applier": "能独立应用", "teacher": "能教别人", "expert": "达到专业水准"}
  MOTIVATION_LABELS = {"interest": "出于个人兴趣", "exam": "为了通过考试", "work": "工作所需", "problem": "为了解决具体问题", "companion": "陪伴孩子学习"}
  PACE_LABELS = {"chill": "慢节奏，重消化", "normal": "稳步推进", "sprint": "高强度冲刺"}
  ```
- 渲染示例：
  > "学生当前水平：**完全陌生**，目标：**能独立应用**，学习动机：**工作所需**（具体原因：下个月接手团队的 Rust 项目）。请按 **稳步推进** 节奏，避免超出当前水平的内容，并在举例时优先选择工作场景。"
- Pydantic 校验：在 `CourseCreate` 处用 `field_validator` 校验 `meta` 中枚举值合法，非法值返回 422。**不要**用 `Literal[...]` 嵌套 dict 强校验（前端扩展时痛苦），自定义 validator 更灵活。

**前端常量**

- `frontend/src/constants/courseDomains.ts` 同时导出枚举 key → 中文标签的映射，避免散落在多个组件。
- 表单提交前做本地校验，但不要硬卡 `motivation_detail / weekly_minutes`（允许跳过）。

## 附录 B · `/persona/generate` 版权角色处理策略

当 `inspiration_type="character"` 时，用户输入可能包含具体作品角色名（"像哈利波特里的卢平教授"、"像鬼灭之刃的炼狱杏寿郎"）。直接复述原作设定有版权风险，且不符合"用户自创"的产品定位。须做**风格借鉴而非角色复制**。

### 1. Prompt 强约束（写入 `PERSONA_GENERATE_PROMPT` 模板）

```
当输入提到具体作品角色（动漫/小说/影视/游戏中的人物）时，遵循"风格借鉴"原则：
- ✅ 可以提取的：性格倾向（温和/严厉/古怪）、说话风格（文白/简练/比喻）、教学态度、典型情绪反应模式
- ❌ 禁止保留的：原作角色姓名、原作专有名词（霍格沃茨/呼吸法/查克拉等）、原作世界观背景、原作具体台词或口头禅、外貌细节复述
- name_suggestion 必须是原创新名字，不得包含原角色名的任何字
- background 必须放置在本系统的"知者"语境中（学院/书院/研究所等通用设定），不得提及原作世界
- 历史人物（孔子、苏格拉底）或公共领域人物（莎士比亚），上述限制放宽，但 name_suggestion 仍略作调整避免完全重名
```

### 2. 输入引导（产品化避雷）

前端"从角色启发"输入框 placeholder 不直接示例版权角色，而是引导描述**特征**：

> placeholder: "描述你想要的老师特质，比如'像一位温和但坚持原则的图书管理员'、'有点神经质但很博学的研究员'……"
>
> 提示文案: "你也可以提到一个你喜欢的角色，AI 会借鉴 TA 的性格风格，但会创造一个全新的人物。"

### 3. 关键词预过滤（二期，第一版不做）

第一版依赖 LLM 自身遵守 prompt 规则。后续若问题频发，加 `backend/services/llm/copyright_keywords.json`：

- 命中时**不拦截**，而是在 prompt 末尾追加："注意：用户输入提到 {keyword}，请特别确保输出不包含此词及其衍生词。"

### 4. 输出二次校验（路由层）

在 [archive.py:1444-1456](backend/api/routes/archive.py#L1444) 解析 LLM 返回 JSON 后：

```python
if req.inspiration_type == "character":
    suspicious = re.findall(r"[\u4e00-\u9fa5]{2,4}", req.description)
    leaked = [t for t in suspicious
              if t in data.get("name_suggestion", "")
              or t in (data.get("background") or "")]
    if leaked:
        data["warnings"] = [f"AI 输出可能包含原作词汇：{', '.join(leaked)}，建议手动调整"]
```

`PersonaGenerateResponse` 加可选字段 `warnings: list[str] | None = None`，软提示不阻塞。

### 5. UI 所有权宣告

Sage Step 5 预览页底部小字：

> "这个角色由 AI 协助你共同创作。你可以随时编辑任何字段，TA 是你的知者。"

法律意义上的用户创作主张，心理学上的归属感强化。

### 6. 不做的事

- ❌ 不维护"禁止角色名单"——维护成本爆炸且文化敏感
- ❌ 不在前端预先检测用户输入文本——破坏自由表达
- ❌ 不让 LLM 拒绝生成——只调整生成结果，永远要给用户一个产出

### 7. 测试用例（加入 `test_persona_generate.py`）

| 输入 | 期望行为 |
|---|---|
| "像哈利波特里的卢平教授" | name_suggestion 不含"卢平"；background 不提霍格沃茨；保留温和+敏感+博学风格 |
| "像孔子那样" | name 可建议"子言""孔门后学"等衍生名；允许提及"周礼""仁"等公共领域概念 |
| "一位严厉但公正的女性物理学家" | 完全自由生成，无需触发任何警告 |
| "像鬼灭之刃的炼狱杏寿郎那种热血" | name 不含"炼狱"；不提"呼吸法""鬼"等原作词；保留热血+坚定风格；warnings 列出"炼狱""杏寿郎" |

## 二、改进方案

### Phase 1: 全局用户层 (UserProfile) — 优先级：高

#### 2.1.1 设计目标
- 记录用户跨世界稳定的学习特征
- 与 world 级别的 LearnerProfile 区分
- 为提示词提供全局上下文

#### 2.1.2 存储设计

**方案A：在 users 表添加 JSON 列（推荐）**

```sql
-- 在 users 表添加
ALTER TABLE users ADD COLUMN profile JSON DEFAULT '{}';
```

**方案B：独立表（更灵活）**

```sql
CREATE TABLE user_profiles (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER UNIQUE REFERENCES users(id) ON DELETE CASCADE,
    profile     JSON NOT NULL DEFAULT '{}',
    created_at  TEXT DEFAULT (datetime('now')),
    updated_at  TEXT DEFAULT (datetime('now'))
);
```

#### 2.1.3 Profile 结构设计

```json
{
  "global_preferences": {
    "visual_examples": {"value": true, "confidence": 0.9, "evidence": "跨世界统计"},
    "analogy_based": {"value": true, "confidence": 0.85, "evidence": "多个世界都有效"},
    "pace": {"value": "slow_deliberate", "confidence": 0.7, "evidence": "大部分世界反馈"},
    "preferred_topics": {"value": ["哲学", "逻辑"], "confidence": 0.8, "evidence": "主动追问的领域"}
  },
  "global_affect": {
    "frustration_tolerance": {"value": "low", "confidence": 0.9},
    "curiosity_baseline": {"value": "high", "confidence": 0.85},
    "encouragement_response": {"value": "strong", "confidence": 0.9}
  },
  "learning_style": {
    "modality": "visual",
    "reflection_habit": "high",
    "question_frequency": "frequent"
  },
  "cross_world_mastery": {
    "total_concepts_learned": 45,
    "total_sessions": 12,
    "average_mastery": 0.65
  }
}
```

#### 2.1.4 数据来源

1. **跨世界汇总**：定期从各 world 的 LearnerProfile 汇总稳定特征
2. **直接录入**：用户主动设置（如"我更喜欢视觉学习"）
3. **LLM推断**：从对话行为模式中推断

#### 2.1.5 提示词注入位置

在 `learning_engine.py` 的 `_build_dynamic_layer()` 中添加：

```python
# 0. Global user profile (before other sections)
if user_profile:
    global_lines = []
    profile = user_profile.get("global_preferences", {})
    if profile:
        pairs = [f"{k}: {v['value']}" for k, v in profile.items() if v.get("value")]
        if pairs:
            global_lines.append(f"用户全局偏好: {', '.join(pairs)}")
    if global_lines:
        parts.insert(0, "【用户全局特征】\n" + "\n".join(global_lines))
```

| 文件 | 改动 |

| `services/learning_engine.py` | 获取 user_profile 并注入提示词 |

### Phase 2: Traveler 信息注入 — 优先级：中

#### 2.2.1 设计目标
- 根据 course_system_design.md，Traveler 是用户在游戏世界中的化身
- 决定是否需要向 AI 暴露用户扮演的角色信息

#### 2.2.2 当前设计决策

根据文档说明：
> "旅者无内心独白——旅者就是用户本人"

**推荐方案：不注入 Traveler 角色设定到提示词**

理由：
1. Traveler 就是用户本身，不需要额外角色扮演
2. 避免 AI 混淆"用户"和"角色"
3. 简化提示词逻辑

#### 2.2.3 如需注入（备选方案）

如果未来需要向 AI 暴露角色信息，在 `_build_static_layer()` 中添加：

```python
def _build_static_layer(self, teacher_persona, traveler_character=None):
    # ... existing persona code ...
    
    if traveler_character:
        parts.append(f"\n【学习者身份】\n用户扮演的角色是 {traveler_character.name}。")
        if traveler_character.background:
            parts.append(f"角色背景: {traveler_character.background}")
```

### Phase 3.5: Mermaid 可视化支持 — 优先级：中

#### 2.3.5 现状分析

| 层面 | 状态 | 说明 |
|------|------|------|
| **提示词注入** | ✅ 有 | 告知 LLM "可以使用 Mermaid" |
| **后端解析** | ❌ 没有 | LLM 返回的 Mermaid 未被验证 |
| **后端渲染** | ❌ 没有 | 依赖前端渲染 |

**当前流程**：
```
LLM 返回 "```mermaid\ngraph LR\n  A --> B\n```"
    ↓
后端透传整个回复给前端
    ↓
前端解析并渲染 Mermaid
```

#### 2.3.5.2 可选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A. 保持现状** | 前端负责渲染，后端只透传 | 简单，无额外开发 | LLM 语法错误时前端渲染失败 |
| **B. 后端验证** | 解析 LLM 返回的 Mermaid，验证语法 | 减少前端错误 | 增加后端复杂度 |
| **C. 后端生成** | 用 LLM 生成 Mermaid，插入到回复中 | 可控制质量 | 需要额外的 LLM 调用 |

#### 2.3.5.3 推荐方案

**推荐方案 A（保持现状）**，原因：
1. 前端已有 Mermaid 渲染库（mermaid.js）
2. LLM 生成 Mermaid 的成功率较高
3. 错误时前端可降级显示代码块

**后续如需增强**，可考虑方案 B：
```python
def validate_mermaid(response: str) -> tuple[bool, str]:
    """验证 Mermaid 语法"""
    import re
    pattern = r'```mermaid\n(.*?)\n```'
    matches = re.findall(pattern, response, re.DOTALL)
    # 可用 mermaid.js 的 CLI 验证
    return len(matches) == 0 or all(validate_syntax(m) for m in matches)
```

### Phase 3.6: Episode Summary 自动生成 — 优先级：中

#### 2.3.6 设计目标

每轮对话后调用 LLM 生成 episode 摘要，解决当前 `retrieve_memories` 返回内容为空的问题。

根据 `course_system_design.md`，episodes 需要以下字段：
- `summary`: 情节摘要
- `significance`: 标记类型（breakthrough/struggle/correction/connection）

#### 2.3.6.2 实现方案

```python
async def _generate_episode_summary(
    self,
    user_msg: str,
    teacher_reply: str,
    emotion: dict,
    session_id: int
) -> dict:
    """LLM 生成 episode 摘要"""
    prompt = f"""根据以下对话生成学习情节摘要。

学生: {user_msg}
教师: {teacher_reply}
情绪: {emotion.get('emotion_type', 'neutral')}

请分析这个对话的关键点，输出 JSON：
{{
  "summary": "情节摘要，1-2句话",
  "significance": "normal|breakthrough|struggle|correction|connection",
  "related_concepts": ["概念1", "概念2"]
}}

significance 说明：
- breakthrough: 学生有重大突破或理解
- struggle: 学生遇到困难
- correction: 学生被纠正错误
- connection: 学生建立新联系
- normal: 普通对话

只输出 JSON，不要其他内容。"""
    
    result = await llm_adapter.chat(messages=[{"role": "user", "content": prompt}])
    return json.loads(result)
```

### Phase 3.9: 模块化提示词注入器 — 优先级：高

#### 2.3.9 设计目标

将 `_build_dynamic_layer()` 拆分为独立的记忆模块，支持灵活组装和扩展。

#### 2.3.9.1 核心价值

| 价值 | 说明 |
|------|------|
| **可维护性** | 各模块独立，新增/修改一个记忆类型不影响其他 |
| **可测试性** | 每个模块可单独单元测试 |
| **可扩展性** | 新增 SceneConfig 支持不同场景（学习/复习/评估） |
| **可复用性** | 模块可在不同场景配置中复用 |

#### 2.3.9.2 架构设计

```
PromptBuilder（主类）
│

# base.py
from abc import ABC, abstractmethod
from typing import TYPE_CHECKING

if TYPE_CHECKING:
    from sqlalchemy.orm import Session

class MemoryModule(ABC):
    """记忆模块基类"""
    
    @abstractmethod
    def get_section_name(self) -> str:
        """模块名称，如【当前知识状态】"""
        pass
    
    @abstractmethod
    def get_priority(self) -> int:
        """组装顺序（越小越靠前）"""
        pass
    
    @abstractmethod
    def should_include(self, context: dict) -> bool:
        """是否包含此模块"""
        pass
    
    @abstractmethod
    def assemble(self, context: dict) -> str | None:
        """
        组装模块内容
        
        Args:
            context: 包含以下键的字典
                - db: Session
                - world_id: int
                - session_id: int
                - learner_profile: LearnerProfile | None
                - user_message: str
                - prev_emotion: dict | None
                - checkpoint_time: str | None
                - ... 其他上下文
        
        Returns:
            组装后的字符串，或 None 表示跳过此模块
        """
        pass


class ContextProvider(ABC):
    """上下文提供者基类（关系阶段、脚手架等）"""
    
    @abstractmethod
    def get_priority(self) -> int:
        pass
    
    @abstractmethod
    def should_include(self, context: dict) -> bool:
        pass
    
    @abstractmethod
    def assemble(self, context: dict) -> str | None:
        pass
```

# modules/knowledge.py
from backend.services.base import MemoryModule
from backend.services.knowledge import knowledge_service

class KnowledgeModule(MemoryModule):
    """知识状态模块 - 按 Bloom 认知层级分组"""
    
    def get_section_name(self) -> str:
        return "【当前知识状态】"
    
    def get_priority(self) -> int:
        return 10
    
    def should_include(self, context: dict) -> bool:
        return True  # 始终包含
    
    def assemble(self, context: dict) -> str | None:
        db = context["db"]
        world_id = context["world_id"]
        checkpoint_time = context.get("checkpoint_time")
        
        graph = knowledge_service.get_knowledge(db, world_id)
        concepts = graph.get("concepts", {})
        
        if not concepts:
            return None
        
        # 按掌握度分组（Bloom 认知层级）
        mastered = []      # mastery >= 0.8 (apply+)
        learning = []     # 0.4 <= mastery < 0.8 (understand)
        new = []          # mastery < 0.4 (remember)
        
        for concept_id, concept in concepts.items():
            if not isinstance(concept, dict):
                continue
            mastery = float(concept.get("mastery", 0.0))
            if mastery >= 0.8:
                mastered.append((concept.get("name", concept_id), mastery))
            elif mastery >= 0.4:
                learning.append((concept.get("name", concept_id), mastery))
            else:
                new.append((concept.get("name", concept_id), mastery))
        
        lines = []
        if mastered:
            names = ", ".join(f"{name}({m:.1f})" for name, m in mastered[:5])
            lines.append(f"已掌握(apply+): {names}")
        if learning:
            names = ", ".join(f"{name}({m:.1f})" for name, m in learning[:5])
            lines.append(f"学习中(understand): {names}")
        if new:
            names = ", ".join(f"{name}({m:.1f})" for name, m in new[:5])
            lines.append(f"初识(remember): {names}")
        
        return "\n".join(lines) if lines else None
```

# modules/preference.py
from backend.services.base import MemoryModule

class PreferenceModule(MemoryModule):
    """学习偏好模块"""
    
    def get_section_name(self) -> str:
        return "【学习偏好】"
    
    def get_priority(self) -> int:
        return 50
    
    def should_include(self, context: dict) -> bool:
        return context.get("learner_profile") is not None
    
    def assemble(self, context: dict) -> str | None:
        profile = context.get("learner_profile")
        if not profile:
            return None
        
        profile_data = profile.profile if isinstance(profile.profile, dict) else {}
        preferences = profile_data.get("preferences", {})
        
        if not preferences or not isinstance(preferences, dict):
            return None
        
        lines = []
        for key, value in preferences.items():
            if isinstance(value, dict):
                val = value.get("value")
                if val is not None:
                    lines.append(f"{key}: {val}")
            elif value:
                lines.append(f"{key}: {value}")
        
        return ", ".join(lines) if lines else None
```

# modules/metacognition.py
from backend.services.base import MemoryModule

class MetacognitionModule(MemoryModule):
    """元认知模块 - MSKT 四维度"""
    
    def get_section_name(self) -> str:
        return "【元认知】"
    
    def get_priority(self) -> int:
        return 70
    
    def should_include(self, context: dict) -> bool:
        return context.get("learner_profile") is not None
    
    def assemble(self, context: dict) -> str | None:
        profile = context.get("learner_profile")
        if not profile:
            return None
        
        profile_data = profile.profile if isinstance(profile.profile, dict) else {}
        metacognition = profile_data.get("metacognition", {})
        
        if not metacognition or not isinstance(metacognition, dict):
            return None
        
        MSKT_DIMENSIONS = ["planning", "monitoring", "regulating", "reflecting"]
        lines = []
        
        for dim in MSKT_DIMENSIONS:
            dim_data = metacognition.get(dim)
            if isinstance(dim_data, dict):
                value = dim_data.get("value")
                if value:
                    lines.append(f"{dim}: {value}")
        
        return ", ".join(lines) if lines else None
```

```python
# builder.py
import logging
from typing import Any

from backend.services.base import ContextProvider, MemoryModule
from backend.services.knowledge import knowledge_service

logger = logging.getLogger(__name__)


class SceneConfig:
    """场景配置"""
    
    LEARNING = "learning"
    REVIEW = "review"
    ASSESSMENT = "assessment"
    
    @classmethod
    def get_modules(cls, scene: str) -> list[type[MemoryModule]]:
        """获取场景对应的模块列表"""
        all_modules = [
            KnowledgeModule,
            SkillModule,
            MisconceptionModule,
            EpisodeModule,
            PreferenceModule,
            AffectModule,
            MetacognitionModule,
            MemoryRetrievalModule,
        ]
        
        if scene == cls.REVIEW:
            return [KnowledgeModule, MemoryRetrievalModule]
        elif scene == cls.ASSESSMENT:
            return [KnowledgeModule, SkillModule]
        else:  # LEARNING
            return all_modules
    
    @classmethod
    def get_contexts(cls, scene: str) -> list[type[ContextProvider]]:
        """获取场景对应的上下文提供者列表"""
        return [RelationshipContext, ScaffoldContext]


class PromptBuilder:
    """提示词构建器"""
    
    def __init__(
        self,
        knowledge_svc=None,
        relationship_svc=None,
    ):
        self.knowledge = knowledge_svc or knowledge_service
        self.relationship = relationship_svc
    
    def build_static_layer(
        self,
        teacher_persona,
        traveler_character=None,
    ) -> str:
        """构建静态层"""
        parts = []
        
        # 1. Teacher Persona
        persona = teacher_persona.system_prompt_template or ""
        if not persona:
            persona = (
                f"你是 {teacher_persona.name}，一位富有耐心的教师，"
                f"运用苏格拉底教学法帮助学生自主思考和探索知识。"
            )
        parts.append(persona)
        
        # 2. Socratic Method Rules
        parts.append("""【教学方法——苏格拉底式提问】
1. 绝不直接给出答案。通过一连串由浅入深的问题引导学习者自己发现答案。
2. 每次回复至少包含一个引导性问题，推动学习者思考下一步。
3. 当学习者的回答有误时，不要直接纠正；而是通过反问让他们重新审视自己的推理。
4. 适时总结学习者已经掌握的部分，再引向尚未理解的部分。
5. 保持专业、耐心、鼓励的态度。""")
        
        # 3. Mermaid Rules
        parts.append("""【可视化工具——Mermaid 图表】
当概念关系、流程步骤、知识结构需要可视化时，可以在回复中使用 ```mermaid 代码块生成图表。
支持的图表类型：flowchart（流程图）、mindmap（思维导图）、sequenceDiagram（时序图）、classDiagram（类图）、graph（关系图）。
使用时机：解释复杂概念关系、展示解题步骤、梳理知识结构时。不要每次回复都画图，只在图表确实有助于理解时使用。""")
        
        # 4. Traveler 角色信息（Phase 3.10）
        if traveler_character:
            parts.append(f"""【学习者身份】
用户扮演的角色是 "{traveler_character.name}"。""")
            if getattr(traveler_character, "background", None):
                parts.append(f"角色背景: {traveler_character.background}")
            if getattr(traveler_character, "personality", None):
                parts.append(f"角色特点: {traveler_character.personality}")
        
        return "\n\n".join(parts)
    
    def build_dynamic_layer(
        self,
        scene: str,
        context: dict,
    ) -> str:
        """
        构建动态层
        
        Args:
            scene: 场景类型（LEARNING/REVIEW/ASSESSMENT）
            context: 上下文字典，包含:
                - db: Session
                - world_id: int
                - session_id: int
                - relationship_stage: str
                - relationship: dict
                - learner_profile: LearnerProfile | None
                - prev_emotion: dict | None
                - mastery_level: int
                - user_message: str
                - checkpoint_time: str | None
        """
        parts = []
        
        # 1. ContextProviders（关系、脚手架）
        for ctx_class in SceneConfig.get_contexts(scene):
            ctx = ctx_class()
            if ctx.should_include(context):
                content = ctx.assemble(context)
                if content:
                    parts.append(content)
        
        # 2. MemoryModules（按 priority 排序）
        for module_class in SceneConfig.get_modules(scene):
            module = module_class()
            if module.should_include(context):
                try:
                    content = module.assemble(context)
                    if content:
                        parts.append(f"{module.get_section_name()}\n{content}")
                except Exception as e:
                    logger.warning(f"Module {module_class.__name__} failed: {e}")
        
        # 3. Knowledge Context（图谱）
        knowledge_context = context.get("knowledge_context")
        if knowledge_context:
            parts.append("【学习者已掌握的知识关系】\n" + knowledge_context)
        
        return "\n\n".join(parts)
    
    def build_with_fallback(
        self,
        scene: str,
        context: dict,
    ) -> str:
        """带降级的构建"""
        try:
            return self.build_dynamic_layer(scene, context)
        except Exception as e:
            logger.warning(f"PromptBuilder failed: {e}, falling back to basic")
            return self._build_basic_dynamic_layer(context)
    
    def _build_basic_dynamic_layer(self, context: dict) -> str:
        """降级版本 - 简单的动态层"""
        parts = []
        
        relationship_stage = context.get("relationship_stage", "stranger")
        parts.append(f"【当前关系阶段：{relationship_stage}】")
        
        emotion = context.get("prev_emotion", {})
        mastery = context.get("mastery_level", 50)
        emotion_type = emotion.get("emotion_type", "neutral")
        parts.append(f"【学习者状态】情感: {emotion_type} | 知识掌握度: {mastery}/100")
        
        return "\n\n".join(parts)
```

#### 2.3.9.7 错误处理与降级策略

| 错误类型 | 处理策略 | 说明 |

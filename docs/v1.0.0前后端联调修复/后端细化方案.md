### Phase 3.5: Mermaid 可视化支持 — 优先级：中

#### 2.3.5 现状分析

| 层面 | 状态 | 说明 |
|------|------|------|
| **提示词注入** | ✅ 有 | 告知 LLM "可以使用 Mermaid" |
| **后端解析** | ❌ 没有 | LLM 返回的 Mermaid 未被验证 |
| **后端渲染** | ❌ 没有 | 依赖前端渲染 |

**当前流程**：
```
LLM 返回 "```mermaid\ngraph LR\n  A --> B\n```"
    ↓
后端透传整个回复给前端
    ↓
前端解析并渲染 Mermaid
```

#### 2.3.5.2 可选方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **A. 保持现状** | 前端负责渲染，后端只透传 | 简单，无额外开发 | LLM 语法错误时前端渲染失败 |
| **B. 后端验证** | 解析 LLM 返回的 Mermaid，验证语法 | 减少前端错误 | 增加后端复杂度 |
| **C. 后端生成** | 用 LLM 生成 Mermaid，插入到回复中 | 可控制质量 | 需要额外的 LLM 调用 |

#### 2.3.5.3 推荐方案

**推荐方案 A（保持现状）**，原因：
1. 前端已有 Mermaid 渲染库（mermaid.js）
2. LLM 生成 Mermaid 的成功率较高
3. 错误时前端可降级显示代码块

**后续如需增强**，可考虑方案 B：
```python
def validate_mermaid(response: str) -> tuple[bool, str]:
    """验证 Mermaid 语法"""
    import re
    pattern = r'```mermaid\n(.*?)\n```'
    matches = re.findall(pattern, response, re.DOTALL)
    # 可用 mermaid.js 的 CLI 验证
    return len(matches) == 0 or all(validate_syntax(m) for m in matches)
```

### Phase 3.6: Episode Summary 自动生成 — 优先级：中

#### 2.3.6 设计目标

每轮对话后调用 LLM 生成 episode 摘要，解决当前 `retrieve_memories` 返回内容为空的问题。

根据 `course_system_design.md`，episodes 需要以下字段：
- `summary`: 情节摘要
- `significance`: 标记类型（breakthrough/struggle/correction/connection）

#### 2.3.6.2 实现方案

```python
async def _generate_episode_summary(
    self,
    user_msg: str,
    teacher_reply: str,
    emotion: dict,
    session_id: int
) -> dict:
    """LLM 生成 episode 摘要"""
    prompt = f"""根据以下对话生成学习情节摘要。

学生: {user_msg}
教师: {teacher_reply}
情绪: {emotion.get('emotion_type', 'neutral')}

请分析这个对话的关键点，输出 JSON：
{{
  "summary": "情节摘要，1-2句话",
  "significance": "normal|breakthrough|struggle|correction|connection",
  "related_concepts": ["概念1", "概念2"]
}}

significance 说明：
- breakthrough: 学生有重大突破或理解
- struggle: 学生遇到困难
- correction: 学生被纠正错误
- connection: 学生建立新联系
- normal: 普通对话

只输出 JSON，不要其他内容。"""
    
    result = await llm_adapter.chat(messages=[{"role": "user", "content": prompt}])
    return json.loads(result)
```

#### 2.3.6.3 集成到 update_after_chat

```python
def update_after_chat(self, db, world_id, user_msg, teacher_reply, emotion, session_id):
    # ... 现有逻辑 ...
    
    # 生成 episode 摘要
    episode_summary = await self._generate_episode_summary(
        user_msg, teacher_reply, emotion, session_id
    )
    
    episode_record = {
        "time": now,
        "type": "chat",
        "emotion": emotion_type,
        "concepts": touched,
        "summary": episode_summary.get("summary", ""),
        "significance": episode_summary.get("significance", "normal"),
        "related_concepts": episode_summary.get("related_concepts", [])
    }
    episodes.append(episode_record)
```

### Phase 3.7: Misconception 追踪 — 优先级：中

#### 2.3.7 设计目标

显式追踪学生错误认知，区别于"不知道"和"错误地知道"。

根据 `learning_memory_theory.md` 的 ITS Perturbation Model：
> "不知道" → 教；"错误地知道" → 先破再立

#### 2.3.7.2 数据结构

```json
{
  "misconception_id": {
    "type": "misconception",
    "name": "误解名称",
    "severity": "minor|moderate|critical",
    "related_concept": "正确概念",
    "content": "误解的详细描述",
    "t_valid": "2026-03-29T16:30",
    "t_invalid": null,
    "corrected_by": null
  }
}
```

#### 2.3.7.3 误解检测触发条件

| 触发条件 | 说明 |
|---------|------|
| 学生表达与已知概念的矛盾 | 检测到错误认知 |
| 多次相同错误 | 确认误解非偶然失误 |
| LLM 判断为误解 | 直接标记 |

#### 2.3.7.4 提示词注入格式

```
【⚠️ 误解】
[moderate] 认为递归一定需要函数自调用（未纠正）
  → 正确认知: 递归可以用栈模拟
```

### Phase 3.9: 模块化提示词注入器 — 优先级：高

#### 2.3.9 设计目标

将 `_build_dynamic_layer()` 拆分为独立的记忆模块，支持灵活组装。

#### 2.3.9.2 架构设计

```
PromptBuilder
├── MemoryModules（记忆模块）
│   ├── KnowledgeModule        # 知识状态（已掌握/学习中/初识）
│   ├── SkillModule           # 程序性技能
│   ├── MisconceptionModule   # 误解追踪（带警告标记）
│   ├── EpisodeModule         # 交互摘要
│   ├── PreferenceModule      # 学习偏好（来自 LearnerProfile）
│   ├── AffectModule          # 情感模式（来自 LearnerProfile）
│   ├── MetacognitionModule   # 元认知（来自 LearnerProfile）
│   └── MemoryRetrievalModule  # 记忆检索
│
├── ContextProviders（上下文提供者）
│   ├── RelationshipContext   # 关系阶段
│   └── ScaffoldContext       # 脚手架
│
└── SceneConfig（场景配置）
    ├── LEARNING              # 学习对话模式
    ├── REVIEW                # 复习模式
    └── ASSESSMENT            # 评估模式
```

#### 2.3.9.3 模块接口设计

```python
class MemoryModule(ABC):
    @abstractmethod
    def get_section_name(self) -> str:
        """模块名称，如【当前知识状态】"""
    
    @abstractmethod
    def assemble(self, context: dict) -> str | None:
        """组装模块内容，返回 None 表示跳过"""
    
    @abstractmethod
    def get_priority(self) -> int:
        """组装顺序（越小越靠前）"""
```

#### 2.3.9.4 注入顺序（按 learning_memory_theory.md §3）

```
【当前关系阶段】← RelationshipContext
【关系维度指令】← RelationshipContext
【脚手架等级】← ScaffoldContext
【学习者身份】← Traveler 角色信息（见 Phase 3.10）
【当前知识状态】← KnowledgeModule
【程序性技能】← SkillModule
【⚠️ 误解】← MisconceptionModule
【近期关键事件】← EpisodeModule
【学习偏好】← PreferenceModule
【情感模式】← AffectModule

【相关学习记忆】← MemoryRetrievalModule
【学习者已掌握的知识关系】← KnowledgeContext（图谱）
```

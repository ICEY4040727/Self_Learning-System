# 记忆系统文件化改造设计

> Issue: #207  
> Date: 2026-04-14  
> Status: Design Phase

## 1. 现状分析

### 1.1 当前模型职责重叠

| 模型 | 表名 | 存储内容 | 实际使用 |
|------|------|----------|----------|
| `Checkpoint` | `checkpoints` | `state` JSON (relationship, course_id, character IDs) | ✅ 活跃 — 6 个 API 端点 |
| `SaveSnapshot` | `save_snapshots` | `payload` JSON (完整存档) | ❌ 死代码 — 零 API/前端调用 |

**关键发现**: `SaveSnapshot` 在迁移 `2026_04_09_add_memory_facts_and_save_snapshots.py` 中创建，但从未被任何路由或前端引用。是纯粹的死代码。

### 1.2 当前 Checkpoint.state 内容

```python
state = {
    "relationship": {...},           # 关系维度
    "course_id": 789,               # 课程 ID
    "sage_character_id": 1,         # 导师角色 ID
    "traveler_character_id": 2,     # 旅人角色 ID

```

这是**最小元数据**，不是完整存档。完整数据散落在 Session + ChatMessage + LearnerProfile 等表中。

### 1.3 现有导出功能

`/checkpoints/{id}/export` 和 `/checkpoints/import` 已能生成/导入 JSON，但：
- 导出是**按需生成**，不持久化到文件
- 不包含 memory_facts 快照
- 不支持时间线过滤

## 2. 设计方案：统一到 Checkpoint + 文件存储

### 2.1 核心决策

| 决策 | 选择 | 理由 |
|------|------|------|
| SaveSnapshot 去留 | **删除** | 死代码，与 Checkpoint 职责重叠 |
| 存储策略 | Checkpoint 表存索引 + JSON 文件存内容 | 满足 Issue 需求 |
| 文件位置 | `data/saves/` | 与 `data/socratic_learning.db` 同级 |
| MemoryFact 时态 | 添加 `t_valid`/`t_invalid`/`checkpoint_id` | 支持分支时间线 |

### 2.2 新 Checkpoint 表结构

```python
class Checkpoint(Base):
    __tablename__ = "checkpoints"

    id = Column(Integer, primary_key=True, index=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    world_id = Column(Integer, ForeignKey("worlds.id", ondelete="CASCADE"), nullable=False)
    session_id = Column(Integer, ForeignKey("sessions.id"), nullable=True)
    save_name = Column(String(100), nullable=False)
    message_index = Column(Integer, nullable=False, default=0)
    
    # === 新增字段 ===
    file_path = Column(String(255), nullable=True)  # JSON 文件相对路径
    file_size_bytes = Column(Integer, nullable=True)  # 文件大小
    
    # === 保留（兼容旧数据） ===
    state = Column(JSON, nullable=False, default=dict)  # 旧存档仍用此字段
    thumbnail_path = Column(String(255), nullable=True)
    created_at = Column(DateTime, default=_utcnow)
```

**兼容策略**: 
- `file_path` 为 NULL → 旧存档，从 `state` 字段读取
- `file_path` 有值 → 新存档，从文件读取完整内容

### 2.4 SaveFileManager 服务

```python
# backend/services/save_file_manager.py

import json
from pathlib import Path
from datetime import UTC, datetime

SAVE_DIR = Path("data/saves")

class SaveFileManager:
    """管理存档 JSON 文件的读写"""
    
    @staticmethod
    def ensure_save_dir(user_id: int) -> Path:
        """确保用户存档目录存在: data/saves/{user_id}/"""
        user_dir = SAVE_DIR / str(user_id)
        user_dir.mkdir(parents=True, exist_ok=True)
        return user_dir
    
    @staticmethod
    def write_save_file(user_id: int, checkpoint_id: int, data: dict) -> str:
        """写入存档文件，返回相对路径"""
        user_dir = SaveFileManager.ensure_save_dir(user_id)
        timestamp = datetime.now(UTC).strftime("%Y%m%d%H%M%S")
        filename = f"checkpoint_{checkpoint_id}_{timestamp}.json"
        filepath = user_dir / filename
        filepath.write_text(json.dumps(data, ensure_ascii=False, indent=2), encoding="utf-8")
        return f"{user_id}/{filename}"  # 相对路径
    
    @staticmethod
    def read_save_file(relative_path: str) -> dict | None:
        """读取存档文件"""
        filepath = SAVE_DIR / relative_path
        if not filepath.exists():
            return None
        return json.loads(filepath.read_text(encoding="utf-8"))
    
    @staticmethod
    def delete_save_file(relative_path: str) -> bool:
        """删除存档文件"""
        filepath = SAVE_DIR / relative_path
        if filepath.exists():
            filepath.unlink()
            return True
        return False
```

### 2.5 新 MemoryFact 时态字段

```python
class MemoryFact(Base):
    __tablename__ = "memory_facts"
    
    # ... 现有字段 ...
    
    # === 新增：时态支持 ===
    t_valid = Column(DateTime, default=_utcnow)       # 记忆生效时间
    t_invalid = Column(DateTime, nullable=True)        # 记忆失效时间（NULL = 仍有效）
    checkpoint_id = Column(Integer, ForeignKey("checkpoints.id"), nullable=True)  # 关联的存档点
```

**时间线过滤逻辑**：
- 加载 checkpoint C 时，查询 `t_valid <= C.created_at AND (t_invalid IS NULL OR t_invalid > C.created_at)`
- 创建分支时，不复制 MemoryFact，而是通过时间线过滤实现隔离

## 4. 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Docker 卷挂载 `data/saves/` | 文件丢失 | docker-compose.yml 添加 volume 映射 |
| 文件与 DB 不一致 | 读档失败 | 事务内写文件，失败回滚；读档时 fallback 到 state 字段 |
| 大量聊天记录导致 JSON 过大 | 性能 | 单文件限制 10MB，超出时截断 chat_history |
| 删除 SaveSnapshot 影响已有 DB | 迁移失败 | 迁移中 DROP TABLE 前确认无数据 |

## 5. 不做清单 (Out of Scope)

- ❌ 多用户文件隔离（已通过 `data/saves/{user_id}/` 实现基础隔离）

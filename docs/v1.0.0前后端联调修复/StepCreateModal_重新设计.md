# StepCreateModal 角色创建向导重新设计

## 一、当前设计问题

当前 StepCreateModal 的"快速创建"过于敷衍：
1. **Step 2 模板选择**只有 4 个泛化的模板名（苏格拉底型/爱因斯坦型等），没有实质性配置
2. **微调滑块**（严谨/温和、快速/细致等）只是调整参数，不会真正影响 AI 扮演
3. **没有对接后端 `TeacherPersona`**——当前创建 sage 角色时虽然会自动创建 TeacherPersona，但没有让用户配置人格

---

## 二、重新设计：四步创建向导

### Step 1：角色定位
**目标**：帮助用户确定知者的"身份定位"

| 问题 | 选项 | 说明 |
|------|------|------|
| 角色原型 | 历史人物 / 虚构角色 / 自定义 | 决定后续提示的倾向性 |
| 领域专长 | 数学/物理/哲学/编程/文学/自定义 | 影响 AI 的知识边界 |
| 角色雏形 | 预设模板 + 自定义描述 | 预设 + 自定义 |

**预设角色模板（可点击选择）**：
```
🏛️ 苏格拉底     古希腊哲学家，擅长反问引导
🎯 诸葛亮        足智多谋，擅长策略性思考启发
📚 爱因斯坦      打破常规，鼓励大胆假设
🤝 探险伙伴      平等同行，共同探索未知
🌸 治愈系学姐   温柔耐心，善于鼓励
```

### Step 2：人格塑造
**目标**：配置 AI 扮演所需的关键人格参数

| 字段 | 输入方式 | 说明 |
|------|----------|------|
| `personality` | 多选标签 + 自定义 | 性格特点：严谨/温和/幽默/耐心/犀利/鼓励/质疑 |
| `background` | 文本输入（2-4句） | 背景故事：角色的来历、经历、为什么会当老师 |
| `speech_style` | 多选标签 + 示例预览 | 说话风格：文绉绉/口语化/专业术语/打比方/反问句 |
| `tags` | 可编辑标签组 | 特质标签：可添加/删除/自定义 |

**人格预览区**：显示 AI 将如何演绎这个角色（基于配置生成示例台词）

### Step 3：AI 优化（可选）
**目标**：让 AI 根据配置生成/优化人格描述

| 功能 | 说明 |
|------|------|
| AI 生成人格 | 根据 Step 1-2 的配置，生成完整的 `system_prompt_template` |
| AI 建议 | 提供 `name_suggestion`、`traits` 补充 |
| 手动编辑 | 生成后可调整 `system_prompt_template` 内容 |

**后端接口**（需确认状态）：
```python
POST /api/persona/generate
Request: { "description": str }
Response: {
    "system_prompt_template": str,
    "traits": ["耐心", "反问", "哲学思维"],
    "name_suggestion": "苏格拉底"
}
```

### Step 4：完成角色
**目标**：基础信息 + 视觉呈现

| 字段 | 输入方式 | 说明 |
|------|----------|------|
| `name` | 文本输入 | 角色名称 |
| `title` | 文本输入 | 称号（如"哲学之父"） |
| `description` | 文本输入 | 人物简介（用于卡片展示） |
| `avatar` | 图片上传 | 角色头像 |
| `color` | 颜色选择器 | 卡片主题色 |

## 三、字段映射汇总

| Character 接口字段 | 来源 | 生成方式 |
|-------------------|------|----------|
| `name` | Step 4 用户输入 | 用户填写 |

| `background` | Step 2 配置 | 用户输入 |

| `tags` | Step 2 配置 | 用户编辑标签 |
| `avatar` | Step 4 上传 | 文件上传 |
| `title` | Step 4 输入 | 用户填写 |
| `description` | Step 4 输入 | 用户填写 |

| `TeacherPersona.system_prompt_template` | Step 3 AI 生成 | AI 根据配置生成 |
| `TeacherPersona.traits` | Step 2 + AI | 标签 + AI 补充 |

## 六、命名对齐问题（待讨论）

当前存在的命名混淆：

| 当前名称 | 位置 | 问题 |
|---------|------|------|
| `Character.personality` | 前端/后端 | 与 `TeacherPersona` 混淆 |
| `TeacherPersona` | 后端模型 | 前端没有对应概念 |
| `system_prompt_template` | 后端字段 | 前端不知道这个字段 |

### 建议方案（待讨论）

**方案 A：统一命名**
- `Character.personality` → `Character.display_personality`（仅用于展示）
- `TeacherPersona` 改名为更直观的名称

**方案 B：职责分离**
- `Character` 仅负责"角色基本信息"（name, avatar, title, color）
- `TeacherPersona` 重命名为 `CharacterPersonality` 或 `RoleIdentity`
- AI 扮演相关全部放在新模型中

**方案 C：保持现状，先完成功能**
- 先实现功能，不做命名重构
- 后续通过文档澄清职责边界

请 Owner 选择方案后实施。
